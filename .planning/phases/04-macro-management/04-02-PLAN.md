  <action>
Add a `MacroEditor` inline component inside `Coach.tsx` (defined before the main `Coach` export function, or inside it as a nested component). The editor appears in the **overview tab** of the Client Detail modal, positioned after the Activity card and before the Weight Trend card (between lines ~674 and ~676).

**MacroEditor component:**

Props: `clientId: string`, `currentTargets: MacroTargets | null` (from Plan 01's expanded type), `coachId: string`, `onSaved: () => void`

State: `calories`, `protein`, `carbs`, `fats` (all string state initialized from `currentTargets` or empty), `isSaving: boolean`

The component renders:
1. A Card with header "SET MACRO TARGETS" (text-sm font-semibold text-muted-foreground)
2. If `currentTargets?.set_by === 'coach'`, show a small badge: "Currently set by coach" in primary/20 colors
3. A 2x2 grid of number inputs: Calories (min 800, max 8000), Protein in g (min 50, max 500), Carbs in g (min 0, max 1000), Fats in g (min 20, max 300)
4. A "Set Macros" button (full width, primary variant, disabled while saving)
5. If `currentTargets?.set_by === 'coach'`, show a "Release to Client" ghost button below the main button

**Save handler (`handleSave`):**
- Validate all 4 fields are non-empty and are valid positive numbers
- If devBypass: call `toast.success('Macros updated (dev mode)')` and `onSaved()`, then return
- Use `getSupabaseClient().from('macro_targets').upsert()` with:
  - `user_id: clientId`
  - `calories: Number(calories)`, `protein: Number(protein)`, `carbs: Number(carbs)`, `fats: Number(fats)`
  - `set_by: 'coach'`, `set_by_coach_id: coachId`
  - `activity_level: 'moderate'` (default for coach-set; avoids NOT NULL violation)
  - `onConflict: 'user_id'`
- On error: `toast.error('Failed to set macros')`
- On success: `toast.success('Macros updated')`, call `onSaved()`
- Track with `analytics.track('coach_set_macros', { clientId })`

**Revert handler (`handleRevert`):**
- Show `window.confirm('Release macro targets back to the client? They will be able to recalculate their own macros.')` -- if cancelled, return
- If devBypass: call `toast.success('Macros released (dev mode)')` and `onSaved()`, then return
- Use `getSupabaseClient().from('macro_targets').update({ set_by: 'self', set_by_coach_id: null }).eq('user_id', clientId)`
- On error: `toast.error('Failed to release macros')`
- On success: `toast.success('Client can now manage their own macros')`, call `onSaved()`
