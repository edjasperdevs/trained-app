**Step 1: Create source icon PNG without rounded corners.**

The existing `public/icon.svg` has `rx="102"` rounded corners that will cause double-rounding on iOS (icon has its own corners + iOS applies superellipse mask). Create a 1024x1024 PNG from the SVG with the `rx` attribute removed.

Approach: Read `public/icon.svg`, modify the SVG string to remove `rx="102"` and `ry="102"` (set both to 0 or remove the attributes), then use sharp (already installed as devDependency) via a one-off Node script to render to 1024x1024 PNG:

```bash
node -e "
const sharp = require('sharp');
const fs = require('fs');
let svg = fs.readFileSync('public/icon.svg', 'utf8');
svg = svg.replace(/rx=\"102\"/g, 'rx=\"0\"').replace(/ry=\"102\"/g, 'ry=\"0\"');
sharp(Buffer.from(svg))
  .resize(1024, 1024)
  .png()
  .toFile('assets/icon-only.png')
  .then(() => console.log('icon-only.png created'))
  .catch(e => console.error(e));
"
```

Create `assets/` directory first if it doesn't exist: `mkdir -p assets`

**Step 2: Create splash source image.**

Create a 2732x2732 splash screen with dark background (#0A0A0A) and the app logo centered at ~20% scale. Use sharp to composite:

```bash
node -e "
const sharp = require('sharp');
const fs = require('fs');
// Read the icon SVG (with or without rounded corners -- splash center logo)
const svg = fs.readFileSync('public/icon.svg', 'utf8');
// Create dark background and composite icon centered
sharp({
  create: {
    width: 2732,
    height: 2732,
    channels: 4,
    background: { r: 10, g: 10, b: 10, alpha: 1 }
  }
})
  .composite([{
    input: Buffer.from(svg),
    gravity: 'centre'
  }])
  .png()
  .toFile('assets/splash.png')
  .then(() => {
    // splash-dark.png is identical (app is always dark themed)
    fs.copyFileSync('assets/splash.png', 'assets/splash-dark.png');
    console.log('splash.png and splash-dark.png created');
  })
  .catch(e => console.error(e));
"
```

If sharp has trouble with the SVG composite, first render the SVG to a ~546px PNG (20% of 2732), then composite that PNG onto the dark background.

Change the function signature from sync to async: `const handleExport = async () => {`

Replace the try/catch block (lines 182-197) with:

```typescript
    try {
      const dataStr = JSON.stringify(exportObj, null, 2)
      const fileName = `trained-backup-${getLocalDateString()}.json`

      if (isNative()) {
        // Write to cache directory (temp, auto-cleaned by OS)
        const result = await Filesystem.writeFile({
          path: fileName,
          data: dataStr,
          directory: Directory.Cache,
          encoding: Encoding.UTF8,
        })

        // Present iOS share sheet with the file
        await Share.share({
          title: 'Trained Backup',
          url: result.uri,
          dialogTitle: 'Export Trained Data',
        })

        // Clean up temp file after sharing
        await Filesystem.deleteFile({
          path: fileName,
          directory: Directory.Cache,
        }).catch(() => {}) // Ignore cleanup errors
      } else {
        // Web: existing Blob + anchor pattern
        const blob = new Blob([dataStr], { type: 'application/json' })
        const url = URL.createObjectURL(blob)
        const link = document.createElement('a')
        link.href = url
        link.download = fileName
        document.body.appendChild(link)
        link.click()
        document.body.removeChild(link)
        URL.revokeObjectURL(url)
      }

      analytics.dataExported()
      toast.success('Data exported')
    } catch (error) {
      toast.error(friendlyError('export your data', error))
    }
```

4. Web path: identical to existing code
5. analytics.dataExported() and toast.success() moved AFTER both branches (outside if/else, inside try)
6. Error handling unchanged
