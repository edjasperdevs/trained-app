  <name>Task 1: Enroll in Apple Developer Program and configure Xcode signing</name>
  <action>
This task requires manual action in Apple's web interfaces and Xcode. Claude cannot perform these steps.

**Step 1: Apple Developer Program Enrollment**
1. Go to https://developer.apple.com/programs/enroll/
2. Sign in with your Apple ID (or create one)
3. Complete the enrollment ($99/year)
4. Wait for identity verification (typically 24-48 hours, sometimes instant)
5. Once approved, note your **Team ID** from https://developer.apple.com/account/#/membership

**Step 2: Update AASA file with real Team ID**
- In `.well-known/apple-app-site-association` (served by Vercel), replace `XXXXXXXXXX` with your actual Team ID
- The app bundle ID is `fitness.welltrained.app`
- Ensure the AASA is served at `https://app.welltrained.fitness/.well-known/apple-app-site-association`

**Step 3: Configure Xcode Signing**
1. Open the Xcode project: `npx cap open ios`
2. Select the "App" target
3. In Signing & Capabilities tab:
   - Check "Automatically manage signing"
   - Select your team (the Apple Developer account you just enrolled)
   - Xcode will create the required provisioning profiles and certificates
4. Verify the bundle identifier is `fitness.welltrained.app`
5. Verify the entitlements include: Associated Domains + APS Environment

**Step 4: Build, Archive, and Upload**
1. Run `npm run build` then `npx cap sync ios` to sync the latest web build
2. In Xcode, select "Any iOS Device" as the build destination
3. Product > Archive
4. Once the archive completes, Xcode Organizer opens
5. Select the archive > Distribute App > App Store Connect > Upload
6. Xcode handles signing with the distribution profile automatically
7. Wait for App Store Connect to process the build (5-15 minutes)

**Step 5: TestFlight Setup**
1. Go to https://appstoreconnect.apple.com/
2. Navigate to your app > TestFlight
3. The uploaded build should appear under "iOS Builds"
4. If prompted, answer the Export Compliance questionnaire:
   - "Does your app use encryption?" > Yes (HTTPS/TLS) but it qualifies for the encryption exemption
   - The app uses standard HTTPS only, no custom encryption
5. Add yourself as an internal tester
6. Install via TestFlight on your device
7. Verify the app works: launch, navigate, sign in, check that push notifications still function

**Note:** If the archive fails with signing errors, ensure:
- Your Apple Developer membership is active
- You selected the correct team in Xcode
- The bundle ID matches `fitness.welltrained.app`
- The push notification capability has a valid provisioning profile
