<task type="auto">
  <name>Task 1: Create delete-account Edge Function</name>
  <files>supabase/functions/delete-account/index.ts</files>
  <action>
Create a new Supabase Edge Function at `supabase/functions/delete-account/index.ts` following the same pattern as `send-push/index.ts` (Deno serve, CORS headers from `../_shared/cors.ts`).

The function must:
1. Handle OPTIONS preflight with corsHeaders
2. Extract Authorization header from the request
3. Create a user-context Supabase client to verify the caller's identity via `getUser()`
4. Return 401 if auth fails
5. Create an admin client using `SUPABASE_SERVICE_ROLE_KEY`
6. Delete storage objects from the `intake-photos` bucket under the user's folder (`user.id/`)
7. Delete from ALL user data tables in dependency-safe order (children before parents):
   - `device_tokens` (user_id)
   - `workout_logs` (user_id)
   - `daily_macro_logs` (user_id)
   - `logged_meals` (user_id)
   - `saved_meals` (user_id)
   - `user_foods` (user_id)
   - `weight_logs` (user_id)
   - `user_xp` (user_id)
   - `weekly_checkins` (client_id)
   - `assigned_workouts` (client_id)
   - `coach_clients` (client_id)
   - `macro_targets` (user_id) -- NOTE: the column is `user_id`, not a foreign key to profiles with CASCADE
   - `profiles` (id) -- last data table, some child tables CASCADE from here but explicit delete is safer
8. Call `supabaseAdmin.auth.admin.deleteUser(user.id)` as the final step
9. Return `{ success: true }` on success, `{ error: message }` with 500 on failure

Use `https://esm.sh/@supabase/supabase-js@2` for the Supabase client import (matching existing Edge Functions).
Use `https://deno.land/std@0.177.0/http/server.ts` for the serve import (matching existing pattern).

Do NOT add `macro_targets` to the CASCADE chain -- the research shows it uses `user_id` without ON DELETE CASCADE from profiles, so explicit deletion is required.
  </action>

<task type="auto">
  <name>Task 2: Add Delete Account button to Settings danger zone</name>
  <files>src/screens/Settings.tsx</files>
  <action>
In `src/screens/Settings.tsx`, add a "Delete Account" button to the existing Danger Zone section, below the existing "Reset All Progress" button. This button should only appear when the user is signed in (`user` is truthy).

Implementation:
1. Add a `handleDeleteAccount` async function:
   - Guard: `if (!user) return`
   - First confirmation via `confirmAction('This will permanently delete your account and ALL associated data from our servers. This action cannot be undone.', 'Delete Account')`
   - On confirm, call `getSupabaseClient().functions.invoke('delete-account')`
   - If error, show `toast.error(friendlyError('delete your account', error))` and return
   - On success, clear all local stores in this order:
     - `useUserStore.getState().resetProgress()`
     - `useXPStore.getState().resetXP()`
     - `useMacroStore.getState().resetMacros()`
     - `useWorkoutStore.getState().resetWorkouts()`
     - `useAvatarStore.getState().resetAvatar()`
     - `useAccessStore.getState().revokeAccess()`
   - Call `await signOut()`
   - Show `toast.success('Account deleted')`
   - Navigate to `/auth`

2. In the Danger Zone expandable section, add the button after the existing "Reset All Progress" button:
   ```tsx
   {user && (
     <Button
       variant="destructive"
       className="w-full"
       onClick={handleDeleteAccount}
     >
       Delete Account
     </Button>
   )}
   ```

3. Update the Danger Zone description text to mention both options:
   "Reset deletes local progress. Delete Account permanently removes your account and all server data."

Import `useAccessStore` if not already imported (check the existing imports from `@/stores`).
  </action>
