<task type="auto">
  <name>Task 2: Create PrivacyInfo.xcprivacy manifest</name>
  <files>ios/App/App/PrivacyInfo.xcprivacy</files>
  <action>
Create `ios/App/App/PrivacyInfo.xcprivacy` with the Apple-required privacy manifest declarations.

This app uses the following APIs that require privacy manifest declarations:
- **NSPrivacyAccessedAPICategoryUserDefaults**: Capacitor and its plugins use UserDefaults for plugin data storage. Reason code: `CA92.1` (access info written by the app itself).

The app does NOT use:
- File timestamp APIs in a way requiring declaration (Filesystem plugin reads/writes files in app container but doesn't use timestamp-specific APIs)
- System boot time APIs
- Disk space APIs

Create the file with this exact content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>NSPrivacyTracking</key>
  <false/>
  <key>NSPrivacyTrackingDomains</key>
  <array/>
  <key>NSPrivacyCollectedDataTypes</key>
  <array/>
  <key>NSPrivacyAccessedAPITypes</key>
  <array>
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>CA92.1</string>
      </array>
    </dict>
  </array>
</dict>
</plist>
```

Notes:
- `NSPrivacyTracking: false` -- the app does not use tracking (Plausible is cookieless/anonymous, Sentry strips PII)
- `NSPrivacyTrackingDomains: []` -- no tracking domains
- `NSPrivacyCollectedDataTypes: []` -- this section is for the plist only; the actual privacy nutrition labels are declared separately in App Store Connect (not in this file)
- The UserDefaults declaration is required because Capacitor core and its plugins (push, preferences, etc.) use UserDefaults

After creating the file, run `npx cap sync ios` to ensure the Xcode project picks up any web build changes. The file itself is placed directly in the Xcode source directory so Xcode will detect it on next open. Note: The file must be added to the Xcode project's build phases (Copy Bundle Resources) -- this typically happens automatically when the file is in the App directory and detected by Xcode. If running `cap sync` does not add it, the user will need to drag it into Xcode's project navigator.
