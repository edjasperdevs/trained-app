  <action>
Refactor `src/screens/Coach.tsx` to replace the local `fetchClients` / `clients` state with `useClientRoster`. This is the primary task of Phase 3.

**Changes to make:**

1. **Replace imports and state:**
   - Import `{ useClientRoster, ClientSummary }` from `@/hooks/useClientRoster`
   - Remove the local `ClientSummary` interface definition (it's now exported from the hook)
   - Remove `const [clients, setClients] = useState<ClientSummary[]>([])`
   - Remove `const [isLoading, setIsLoading] = useState(true)`
   - Remove `const [error, setError] = useState<string | null>(null)`
   - Remove the entire `fetchClients` async function
   - Add: `const { clients, totalCount, page, search, isLoading, error, totalPages, setPage, setSearch, refresh } = useClientRoster()`

2. **Add search input to the header area:**
   - Below the header div (after "Coach Dashboard" title area), add a search input
   - Use the existing `Input` component (already imported)
   - Placeholder: "Search clients..."
   - Value bound to `search`, onChange calls `setSearch(e.target.value)`
   - Add a Search icon from lucide-react (import `{ Search }` from `lucide-react`)
   - Wrap in a relative div with the Search icon positioned absolute left
   - Only show search input when `totalCount > 0` OR `search` is non-empty (don't show on empty roster)

3. **Update the client count display:**
   - Change from `{clients.length} client{clients.length !== 1 ? 's' : ''}` to `{totalCount} client{totalCount !== 1 ? 's' : ''}`
   - This shows total across all pages, not just current page

4. **Update quick stats:**
   - The quick stats (Active Today, Need Check-in, Falling Off) currently iterate `clients` which will now be only the current page. This is acceptable for now -- the stats show page-level activity which is still useful when browsing. Keep the existing logic but operating on the current `clients` array. Alternative: remove the quick stats section entirely if it feels misleading. Use your judgment -- if the page size is 20 and the coach has 100 clients, showing "Active Today: 5" for the current page of 20 is misleading. **Decision: hide the quick stats section when totalPages > 1 to avoid confusion. Show them only for single-page rosters where they're accurate.**

5. **Update client list rendering:**
   - Remove the `.sort()` call on clients (server already sorts alphabetically)
   - Keep the existing card rendering, but update the right-side data to be more informative:
     - Line 1: Show `{client.workouts_last_7_days || 0} workouts` (7d) instead of just level
     - Line 2: Keep `{client.current_streak || 0} day streak`
   - Keep the status emoji indicator (getStatusEmoji)
   - Keep the click handler to open detail modal

6. **Add pagination controls at the bottom of the client list:**
   - Only render when `totalPages > 1`
   - Layout: flex row with justify-between, items-center
   - Left: "Previous" button (Button variant="ghost" size="sm"), disabled when `page === 0`
   - Center: `{page + 1} of {totalPages}` in text-sm text-muted-foreground
   - Right: "Next" button (Button variant="ghost" size="sm"), disabled when `page >= totalPages - 1`
   - Previous onClick: `setPage(page - 1)`
   - Next onClick: `setPage(page + 1)`

7. **Handle empty search results vs empty roster:**
   - When `clients.length === 0` and `search.trim()` is non-empty: show "No clients match your search" message with a "Clear search" button that calls `setSearch('')`
   - When `clients.length === 0` and `search.trim()` is empty: show the existing "No clients yet" empty state with invite button

8. **Update references:**
   - Replace `fetchClients()` calls in `handleInviteClient` and `handleRemoveClient` with `refresh()`
   - The `useEffect(() => { fetchClients(); fetchInvites() }, [])` should become just `useEffect(() => { fetchInvites() }, [])` since useClientRoster handles its own initial fetch
   - Keep all invite-related state and functions unchanged (invites, fetchInvites, handleInviteClient, showAddClient modal, etc.)

9. **Keep everything else unchanged:**
   - Client detail modal (selectedClient, tabs, useClientDetails, WeightChart, ClientMacroAdherence, ClientActivityFeed) -- all stays the same
   - Invite modal -- stays the same
   - Invite status list -- stays the same
   - Analytics tracking -- stays the same
   - handleRemoveClient -- stays the same (but uses `refresh()` instead of `fetchClients()`)
