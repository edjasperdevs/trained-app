Create `src/hooks/useClientRoster.ts` that encapsulates paginated, searchable client roster fetching.

**Hook interface:**
```typescript
const PAGE_SIZE = 20

interface UseClientRosterReturn {
  clients: ClientSummary[]
  totalCount: number
  page: number
  search: string
  isLoading: boolean
  error: string | null
  totalPages: number
  setPage: (page: number) => void
  setSearch: (search: string) => void
  refresh: () => void
}
```

**Implementation details:**

1. Import the `ClientSummary` interface. Since Coach.tsx currently defines it locally, define it in the hook file and export it. The interface matches the existing one in Coach.tsx exactly (all fields from coach_client_summary view).

2. **Server-side fetch function** (`fetchRosterPage`):
   - Use `getSupabaseClient().from('coach_client_summary').select('*', { count: 'estimated' })`
   - Apply `.order('username', { ascending: true, nullsFirst: false })` for alphabetical sort
   - If `search.trim()` is non-empty, apply `.or('username.ilike.%${search}%,email.ilike.%${search}%')`
   - Apply `.range(from, to)` where `from = page * PAGE_SIZE`, `to = from + PAGE_SIZE - 1`
   - Return `{ clients: data || [], totalCount: count || 0 }`

3. **Dev bypass path:**
   - When `devBypass` is true, filter `getMockClients()` by search term (case-insensitive match on username and email)
   - Slice the filtered array for pagination: `filtered.slice(page * PAGE_SIZE, (page + 1) * PAGE_SIZE)`
   - Return the slice and `filtered.length` as totalCount

4. **Debounced search** (match FoodSearch.tsx pattern):
   - Use `useRef<NodeJS.Timeout | null>(null)` for debounce timer
   - The `setSearch` function updates the search state immediately (for controlled input) but debounces the actual fetch by 400ms
   - When search changes, reset page to 0 before fetching
   - Clear debounce timer on unmount

5. **State management:**
   - `useState` for: clients, totalCount, page, search, debouncedSearch, isLoading, error
   - The actual fetch triggers on `page` or `debouncedSearch` changes (useEffect)
   - `totalPages` is a computed value: `Math.max(1, Math.ceil(totalCount / PAGE_SIZE))`
   - `refresh()` re-fetches the current page with current search

6. **Error handling:**
   - Match Coach.tsx pattern: try/catch with network error detection
   - Set `error` state on failure, `null` on success
   - Log errors to console with `[ClientRoster]` prefix

7. **Initial load:**
   - Fetch page 0 with empty search on mount (via useEffect)

Export both the hook and the `ClientSummary` interface. Also export `PAGE_SIZE` constant.
