    Add `data-sentry-mask` attribute to wrapper elements containing health/fitness PII. The attribute triggers Sentry's replay masking (configured in 04-01 via `mask: ['[data-sentry-mask]']`).

    **What IS PII (mask these):** body weight, goal weight, weight history/chart, macro numbers (protein/calories/carbs/fats), meal names, food names, food macros, ingredient data, email address, BMR/TDEE values, body metrics, coach client health data.

    **What is NOT PII (do NOT mask):** XP, streaks, badges, workout exercise names, set/rep counts, avatar stage, screen titles, navigation, general UI chrome.

    **Strategy:** Add `data-sentry-mask` to the NEAREST common wrapper `<div>` or `<section>` that contains the PII cluster. Do NOT add it to every individual number -- find the parent container. This keeps the code clean and catches any child elements.

    **1. src/screens/Settings.tsx:**
    - Find the weight logging section (weight input, current weight display, weight history chart, goal weight) -- add `data-sentry-mask` to its container div
    - Find the email display -- add `data-sentry-mask` to the element showing user email
    - Find body metrics section (if BMR/TDEE/rate of change displayed) -- add `data-sentry-mask` to its container

    **2. src/screens/Macros.tsx:**
    - Find the macro summary/progress section (protein, calories, carbs, fats numbers) -- add `data-sentry-mask` to the container holding all macro progress bars/numbers
    - The meal plan, food search, and meal builder components render inside Macros -- if they render inline (not in separate route), find their wrapper and add `data-sentry-mask`
    - The entire macro tracking content area is PII -- consider adding `data-sentry-mask` to the top-level screen content div (below the header/tabs)

    **3. src/screens/Onboarding.tsx:**
    - Find the weight input step -- add `data-sentry-mask` to the form/section containing weight input
    - Find any goal weight or body metrics input steps -- mask those too

    **4. src/screens/Home.tsx:**
    - Find the macro progress summary (today's protein/calories display) -- add `data-sentry-mask` to its container
    - Find the weight display if shown on home -- mask it
    - Do NOT mask: XP, streaks, avatar, workout summary

    **5. src/screens/Coach.tsx:**
    - Find client macro adherence section -- add `data-sentry-mask` to the container
    - Find client activity feed with health data -- add `data-sentry-mask` to the container
    - If Coach shows client weight/body data -- mask those sections

    **Implementation pattern:**
    ```tsx

    <div className="weight-section">
      <span>{currentWeight} lbs</span>
    </div>

    // AFTER
    <div className="weight-section" data-sentry-mask>
      <span>{currentWeight} lbs</span>
    </div>
    ```

    Use bare `data-sentry-mask` (no value needed) -- Sentry checks for attribute presence.
