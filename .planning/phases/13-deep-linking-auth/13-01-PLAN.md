    **1. Create AASA file at `public/.well-known/apple-app-site-association`:**
    ```json
    {
      "applinks": {
        "details": [
          {
            "appIDs": ["TEAMID.fitness.welltrained.app"],
            "components": [
              {
                "/": "/reset-password*",
                "comment": "Password reset deep links"
              }
            ]
          }
        ]
      }
    }
    ```

    **IMPORTANT:** The `TEAMID` placeholder MUST be replaced with the actual Apple Developer Team ID. Before writing this file, ask the user for their Team ID (found at Apple Developer Portal -> Membership -> Team ID). If the user doesn't have an Apple Developer account yet, use a placeholder like `XXXXXXXXXX` and note it must be updated before deployment.

    Use the modern `appIDs` (plural) + `components` format (iOS 13+). Only scope to `/reset-password*` -- do NOT use a catch-all `*` pattern as that would hijack the PWA web experience for all URLs.

    **2. Update `vercel.json`:**
    - Add an explicit rewrite rule for the AASA path BEFORE the SPA catch-all rewrite, so Vercel serves the actual file instead of index.html:
      ```json
      { "source": "/.well-known/apple-app-site-association", "destination": "/.well-known/apple-app-site-association" }
      ```
    - Update the SPA catch-all rewrite to exclude `.well-known`:
      ```json
      { "source": "/((?!assets|sw|workbox|manifest|\\.well-known).*)", "destination": "/" }
      ```
    - Add a headers entry for AASA content-type:
      ```json
      {
        "source": "/.well-known/apple-app-site-association",
        "headers": [
          { "key": "Content-Type", "value": "application/json" }
        ]
      }
      ```

    **3. Create `ios/App/App/App.entitlements`:**
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>com.apple.developer.associated-domains</key>
        <array>
            <string>applinks:app.welltrained.fitness</string>
        </array>
    </dict>
    </plist>
    ```

    Then add the entitlements file reference to the Xcode project. Check `ios/App/App.xcodeproj/project.pbxproj` for the existing build configuration sections and add `CODE_SIGN_ENTITLEMENTS = App/App.entitlements;` to both Debug and Release build settings for the App target.
  </action>

  <action>
    **1. Create `src/lib/deep-link.ts` -- Central deep link handler:**

    Export a function `initDeepLinkHandler(navigate: (path: string) => void)` that:
    - Imports `App` from `@capacitor/app` and `supabase` from `@/lib/supabase`
    - Imports `isNative` from `@/lib/platform` -- only registers listener if `isNative()` is true
    - Registers `App.addListener('appUrlOpen', ...)` listener
    - On `appUrlOpen`, parses the URL using `new URL(event.url)`
    - Extracts hash fragment params via `new URLSearchParams(url.hash.substring(1))`
    - If `access_token` AND `refresh_token` exist in hash params:
      - Calls `await supabase?.auth.setSession({ access_token, refresh_token })`
      - If `type` param is `'recovery'`, calls `navigate('/reset-password')`
      - Otherwise navigates to `'/'` (generic auth completion)
    - If no auth tokens but URL has a pathname other than `/`, calls `navigate(url.pathname)` for future deep link support
    - Also handles cold start: calls `App.getLaunchUrl()` and if a URL exists, processes it through the same handler logic (extract into a shared `handleDeepLink(url: string, navigate)` function to avoid duplication)

    **2. Create `src/screens/ResetPassword.tsx` -- Password reset form:**

    A screen component matching the app's existing Auth screen styling (dark theme, Card layout). Must:
    - Import `supabase` from `@/lib/supabase`, `useNavigate` from `react-router-dom`, `toast` from `@/stores`
    - Use existing UI components: `Card`, `CardContent`, `CardHeader`, `CardTitle`, `CardDescription`, `Button`, `Input`, `Label`, `Alert`, `AlertDescription` (same imports as Auth.tsx)
    - Show a form with "New Password" and "Confirm Password" fields (both with show/hide toggle using Eye/EyeOff icons, same pattern as Auth.tsx)
    - Validate: password >= 6 chars, passwords match
    - On submit, call `await supabase?.auth.updateUser({ password: newPassword })`
    - On success: show `toast.success('Password updated successfully')`, then navigate to `/` after a brief delay (1 second)
    - On error: show error in an Alert variant="destructive"
    - Include loading state on the submit button
    - Match the centered layout of Auth.tsx: `min-h-screen flex flex-col items-center justify-center px-5`, `w-full max-w-md`
    - Include the WELLTRAINED logo header (same as Auth.tsx)

    **3. Fix `src/stores/authStore.ts` -- redirectTo URL:**

    In the `resetPassword` method, replace:
    ```typescript
    redirectTo: `${window.location.origin}/reset-password`
    ```
    With:
    ```typescript
    redirectTo: 'https://app.welltrained.fitness/reset-password'
    ```
    This is critical because `window.location.origin` returns `capacitor://localhost` in the native app context, which Supabase cannot redirect to and is not a valid Universal Link domain. Hardcode the web domain so the redirect works for both web and native users.

    **4. Wire up in `src/App.tsx`:**

    - Add lazy import for ResetPassword: `const ResetPassword = lazy(() => import('@/screens/ResetPassword').then(m => ({ default: m.ResetPassword })))`
    - Add the `/reset-password` route in the authenticated routes section (the main return with Navigation), BEFORE the `*` catch-all NotFound route:
      ```tsx
      <Route path="/reset-password" element={<Suspense fallback={<HomeSkeleton />}><ResetPassword /></Suspense>} />
      ```
    - ALSO add `/reset-password` to the unauthenticated routes section (where `!user` shows Auth). The user arrives via a recovery deep link which sets the session, but there's a brief moment where the route renders before `onAuthStateChange` fires. Add it alongside the existing `/auth` and `/coach` routes:
      ```tsx
      <Route path="/reset-password" element={<Suspense fallback={<HomeSkeleton />}><ResetPassword /></Suspense>} />
      ```
    - Add deep link initialization in a new `useEffect` inside `AppContent`:
      ```typescript
      import { useNavigate } from 'react-router-dom'
      import { initDeepLinkHandler } from '@/lib/deep-link'

      // Inside AppContent:
      const navigate = useNavigate()

      useEffect(() => {
        initDeepLinkHandler(navigate)
      }, [navigate])
      ```
      Place this useEffect early, before the auth loading check, so deep links are handled even during initialization. The `initDeepLinkHandler` function internally checks `isNative()` and is a no-op on web.
  </action>
