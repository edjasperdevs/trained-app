  <action>
  The goal: Each screen should have AT MOST 2 uppercase elements -- the primary page heading (h1) and at most one section header. Everything else uses weight-driven hierarchy (font-semibold, font-bold) instead of uppercase.

  **Rule for what KEEPS uppercase:**
  1. Page-level h1 titles (e.g., "WORKOUTS", "HOME", "ACHIEVEMENTS") -- these are the screen identity
  2. Section headers that act as category labels (e.g., "CLOSEST TO UNLOCK", "RECENT WORKOUTS") -- small text-sm/text-xs with text-text-secondary, these are structural dividers
  3. Primary CTA button text (already handled by Button.tsx CVA variant -- leave Button.tsx alone)

  **Rule for what LOSES uppercase:**
  - Card titles and names (workout names, badge names, exercise names, stat values)
  - Labels inside cards (protein, calories, streak counts, XP values)
  - Navigation tab labels (in Navigation.tsx -- remove uppercase from tab text)
  - Option labels in selection cards (Onboarding step options like "3 Days", "Push/Pull/Legs")
  - Sub-headings inside cards
  - Avatar evolution stage names
  - Stat label text (e.g., "LEVEL", "STREAK", "XP") -- these become regular weight-driven text
  - Form labels in Onboarding
  - Badge rarity labels (keep colored but remove uppercase)
  - Inline emphasis text (e.g., "Full Compliance Bonus!", "Personal Record")

  **Implementation approach:**
  1. In `src/index.css`, update the global heading styles (around line 304-316): Keep `text-transform: uppercase` on `h1` only. Remove it from `h2`, `h3` (they get hierarchy from font-weight and font-size alone).
  2. For each screen/component file: Search for `uppercase` in className strings. For each occurrence, decide based on the rules above:
     - If it's a page h1 or a section header label → KEEP uppercase
     - If it's anything else → REMOVE `uppercase` and `tracking-wide`/`tracking-wider`/`tracking-widest`. Keep `font-heading` if it's a heading, keep `font-semibold`/`font-bold` for emphasis.
  3. Also remove `font-heading` from non-heading elements (card labels, stat values, option labels) -- `font-heading` (Oswald) should only be used on actual headings, not on every bold text.

  **Per-screen max uppercase target (approximate):**
  - Home: h1 "Welcome" line + section headers like "Today's Quests" = ~2
  - Workouts: h1 "Workouts" + section header "Recent Workouts" = ~2
  - Achievements: h1 "Achievements" + section header "CLOSEST TO UNLOCK" = ~2
  - AvatarScreen: h1 "Your Avatar" + section headers = ~2
  - Settings: h1 "Settings" + section headers = ~2
  - Onboarding: h1 per step + 0-1 section headers = ~1-2
  - AccessGate: h1 "TRAINED" + section header = ~2
  - Components (Badges, StreakDisplay, etc.): 0-1 per component (section labels only)
  </action>

  - Navigation.tsx has 0 uppercase (tab labels are lowercase)
  - App still builds: `npx tsc --noEmit` passes
  </verify>
