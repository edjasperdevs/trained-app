Create `supabase/migrations/007_weekly_checkins.sql` with the full schema from 06-RESEARCH.md:

1. Create `checkin_status` ENUM: 'submitted', 'reviewed'
2. Create `weekly_checkins` table with:
   - id (UUID PK), created_at, updated_at
   - client_id (FK profiles), coach_id (FK profiles, ON DELETE SET NULL)
   - week_of (DATE), status (checkin_status DEFAULT 'submitted')
   - 16 client-submitted fields (see research): water_intake TEXT, caffeine_intake TEXT, hunger_level INTEGER CHECK 1-5, slip_ups TEXT, refeed_date DATE, digestion TEXT, training_progress TEXT, training_feedback TEXT, recovery_soreness TEXT, sleep_quality INTEGER CHECK 1-5, sleep_hours DECIMAL(3,1), stress_level INTEGER CHECK 1-5, stressors TEXT, mental_health TEXT, injuries TEXT, cycle_status TEXT, side_effects TEXT, bloodwork_date DATE, open_feedback TEXT

   - Coach response fields: coach_response TEXT, reviewed_at TIMESTAMPTZ
   - UNIQUE(client_id, week_of)
3. Create indexes: idx_weekly_checkins_client (client_id, week_of DESC), idx_weekly_checkins_coach_pending (coach_id, status, created_at WHERE status = 'submitted')
4. Enable RLS with 4 policies:
   - Clients INSERT own (client_id = auth.uid())
   - Clients SELECT own (client_id = auth.uid())
   - Coaches SELECT via coach_clients join (status = 'active')
   - Coaches UPDATE via coach_clients join (status = 'active')
5. Add Client UPDATE policy: clients can update their own check-ins WHERE status = 'submitted' (allows re-submission before coach reviews)
6. updated_at trigger using existing update_updated_at() function
