<task type="auto">
  <name>Task 1: Add Supabase API runtime caching and fix viewport meta</name>
  <files>vite.config.ts, index.html</files>
  <action>
**vite.config.ts:** Add a Supabase REST API runtime caching rule to the existing `runtimeCaching` array in the workbox config. Place it AFTER the existing USDA and OpenFoodFacts rules.

```typescript
{
  // Supabase REST API - NetworkFirst for fresh data with offline fallback
  urlPattern: /^https:\/\/.*\.supabase\.co\/rest\/v1\/.*/,
  handler: 'NetworkFirst',
  options: {
    cacheName: 'supabase-api-cache',
    networkTimeoutSeconds: 3,
    expiration: {
      maxEntries: 50,
      maxAgeSeconds: 60 * 60 * 24 // 24 hours
    },
    cacheableResponse: {
      statuses: [0, 200]
    }
  }
}
```

IMPORTANT: Do NOT add a rule for `/auth/v1/` endpoints. Auth tokens are sensitive and time-bound -- caching them causes auth failures.

Also add Google Fonts runtime caching rules (two rules: one for fonts.googleapis.com CSS, one for fonts.gstatic.com font files). Both should use CacheFirst with 1-year expiration.

```typescript
{
  urlPattern: /^https:\/\/fonts\.googleapis\.com\/.*/i,
  handler: 'CacheFirst',
  options: {
    cacheName: 'google-fonts-cache',
    expiration: {
      maxEntries: 10,
      maxAgeSeconds: 60 * 60 * 24 * 365
    },
    cacheableResponse: {
      statuses: [0, 200]
    }
  }
},
{
  urlPattern: /^https:\/\/fonts\.gstatic\.com\/.*/i,
  handler: 'CacheFirst',
  options: {
    cacheName: 'gstatic-fonts-cache',
    expiration: {
      maxEntries: 30,
      maxAgeSeconds: 60 * 60 * 24 * 365
    },
    cacheableResponse: {
      statuses: [0, 200]
    }
  }
}
```

**index.html:** Fix the viewport meta tag. Replace:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />
```
With:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

This removes `maximum-scale=1.0` and `user-scalable=no` which fail Lighthouse Accessibility ("Zooming and scaling is not disabled").

Also add `touch-action: manipulation` to the body element in `src/index.css` to prevent double-tap zoom without failing accessibility. Add it to the existing `body` rule:
```css
body {
  /* existing properties... */
  touch-action: manipulation;
}
```

<task type="auto">
  <name>Task 2: Fix color contrast for WCAG AA compliance</name>
  <files>src/themes/trained.ts, src/index.css</files>
  <action>
Fix color tokens that fail WCAG AA contrast requirements. The colors are defined in TWO places that must stay in sync:
1. `src/themes/trained.ts` (runtime theme tokens -- the source of truth, applied via ThemeProvider)
2. `src/index.css` `:root` block (CSS fallback defaults before ThemeProvider mounts)

**Colors to fix (all on dark backgrounds #0A0A0A or #141414):**

1. **colorTextAccent** (currently `#8B1A1A` = ~2.7:1 on #0A0A0A): Change to `#D4443B` (~5.0:1). This is used extensively as text color via Tailwind `text-accent-primary` class. Must remain visually "blood red" but lighter for readability.

2. **colorPrimary** (currently `#8B1A1A`): Change to `#C13A33` (~4.5:1 on #0A0A0A). Used for buttons, borders, glows. When used as text (e.g., `text-accent-primary` maps to accent.primary which is `var(--color-primary)`), it must meet 4.5:1. Keep it in the red family.

3. **colorPrimaryHover** (currently `#A52222`): Change to `#D4443B` (~5.0:1). Hover state should be lighter than primary.

4. **colorSuccess** (currently `#2D5A27` = ~2.3:1 on #141414): Change to `#4CAF50` (~5.3:1 on #141414). Used as text via `text-accent-success`. Must remain "green" but lighter.

5. **colorWarning** (currently `#8B6914` = ~3.0:1 on #141414): Change to `#D4A843` (~6.0:1 on #141414). Used as text via `text-accent-warning`. Must remain "gold/amber" but lighter.

6. **colorError** (currently `#8B1A1A`): Change to same as colorPrimary `#C13A33` for consistency.

7. **colorXPBar** and **colorStreakActive**: Change to same as colorPrimary `#C13A33`.

8. **colorPrimaryMuted**: Update to `rgba(193,58,51,0.15)` to match new primary.

9. **shadowGlow**: Update to `0 0 20px rgba(193,58,51,0.2)` to match new primary.

**Update `src/themes/trained.ts`** tokens object with all the above values.

**Update `src/index.css`** `:root` CSS variables to match the new token values exactly. These are the fallback defaults.
