Create weeklyReportStore with Zustand persist middleware:

**State:**
- `lastShownWeekStart: string | null` - ISO date string of the Sunday when report was last shown (prevents showing same week twice)
- `lastShownDate: string | null` - ISO date string when report was last shown

**Actions:**
- `getWeeklyStats(): WeeklyStats` - aggregates data from dpStore.dailyLogs and workoutStore.workoutLogs for the past 7 days:
  - `dpEarned: number` - sum of daily DP totals for the week
  - `compliancePercentage: number` - (days with DP action / 7) * 100
  - `streak: number` - current obedienceStreak from dpStore
  - `workoutsCompleted: number` - count of completed workouts in date range
  - `proteinDaysHit: number` - count of days where protein > 0 in dailyLogs
  - `bestWorkoutDP: number` - highest single day training DP
- `shouldShowReport(): boolean` - returns true if today is Sunday AND lastShownWeekStart !== current week's Sunday
- `markReportShown(): void` - sets lastShownWeekStart to current week's Sunday, lastShownDate to today

**helpers:**
- `getWeekStartDate(): string` - returns ISO date string for most recent Sunday (week start)

Create highlights.ts utility:

```typescript
export interface Highlight {
  type: 'protein' | 'pr' | 'streak' | 'workouts' | 'compliance' | 'dp'
  title: string
  description: string
  icon: string // Lucide icon name
}

export function generateHighlights(stats: WeeklyStats, longestStreak: number): Highlight[]
```

Highlight generation rules:
1. `protein`: If proteinDaysHit >= 5, "Protein Protocol Mastered - Hit protein target X/7 days"
2. `streak`: If current streak >= 7, "Week-Long Streak - X consecutive days of discipline"
3. `streak`: If current streak > longestStreak (new record), "New Streak Record - Beat your previous best"
4. `workouts`: If workoutsCompleted >= 4, "Training Machine - Completed X workouts this week"
5. `compliance`: If compliancePercentage === 100, "Perfect Week - 100% protocol compliance"
6. `dp`: If dpEarned >= 500, "DP Domination - Earned X+ Discipline Points"

Always return at least one highlight. If no specific milestones hit, return a generic "Week Complete" highlight with the user's top stat.

Export WeeklyStats interface and useWeeklyReportStore from index.ts.
