  <name>Task 1: Create APNs shared helper with JWT caching and HTTP/2 delivery</name>
  <files>
    supabase/functions/_shared/apns.ts
  </files>
  <action>
    Create supabase/functions/_shared/apns.ts:

    1. Import `SignJWT` and `importPKCS8` from `https://esm.sh/jose@5`

    2. Define constants:
       - `APNS_PROD = 'https://api.push.apple.com'`
       - `APNS_DEV = 'https://api.development.push.apple.com'`

    3. Module-level JWT cache: `let cachedJWT: { token: string; expiresAt: number } | null = null`
       - Warm Edge Function instances reuse this across invocations
       - Cold starts regenerate

    4. `async function getAPNsJWT(): Promise<string>`
       - Check cache: if `cachedJWT` exists and `Date.now() < cachedJWT.expiresAt`, return cached token
       - Read secrets from Deno.env: `APNS_KEY_ID`, `APNS_TEAM_ID`, `APNS_P8_KEY`
       - The .p8 key is stored as a single-line secret with `\n` literals -- replace `\\n` with actual newlines: `p8Key.replace(/\\n/g, '\n')`
       - Import private key: `await importPKCS8(p8Key, 'ES256')`
       - Create JWT: `new SignJWT({}).setProtectedHeader({ alg: 'ES256', kid: keyId }).setIssuer(teamId).setIssuedAt().sign(privateKey)`
       - Cache with 50-minute expiry (APNs JWTs valid for 60 min): `{ token: jwt, expiresAt: Date.now() + 50 * 60 * 1000 }`

    5. Export `async function sendAPNs(deviceToken: string, payload: { title: string; body: string; data?: Record<string, string> }): Promise<{ success: boolean; status: number }>`
       - Get JWT via `getAPNsJWT()`
       - Read bundle ID from env: `Deno.env.get('APNS_BUNDLE_ID') || 'fitness.welltrained.app'`
       - Determine host: `Deno.env.get('APNS_ENV') === 'development' ? APNS_DEV : APNS_PROD`
       - POST to `${host}/3/device/${deviceToken}` with headers:
         - `authorization: bearer ${jwt}`
         - `apns-topic: ${bundleId}`
         - `apns-push-type: alert`
         - `apns-priority: 10`
         - `apns-expiration: 0` (immediate delivery, no store-and-forward)
       - Body JSON: `{ aps: { alert: { title, body }, sound: 'default', badge: 1 }, ...(payload.data || {}) }`
       - Note: the `data` fields (like `route`) are placed at the TOP level of the payload (not inside `aps`) so they appear in `notification.data` on the client side
       - Return `{ success: response.ok, status: response.status }`
  </action>

<task type="auto">
  <name>Task 2: Create send-push Edge Function with webhook routing and coach-action filtering</name>
  <files>
    supabase/functions/send-push/index.ts
  </files>
  <action>
    Create supabase/functions/send-push/index.ts following the pattern from send-invite/index.ts:

    1. Import `corsHeaders` from `'../_shared/cors.ts'`, `createClient` from `'https://esm.sh/@supabase/supabase-js@2'`, `sendAPNs` from `'../_shared/apns.ts'`

    2. Define `WebhookPayload` interface:
       ```typescript
       interface WebhookPayload {
         type: 'INSERT' | 'UPDATE' | 'DELETE'
         table: string
         schema: string
         record: Record<string, unknown>
         old_record: Record<string, unknown> | null
       }
       ```

    3. Create `getNotificationDetails(payload: WebhookPayload)` that returns `{ clientId, title, body, route } | null`:
       - Switch on `payload.table`:
       - `'assigned_workouts'` (INSERT): return `{ clientId: record.client_id, title: 'New Workout Assigned', body: 'Your coach assigned a new workout', route: '/workouts' }`
       - `'macro_targets'` (UPDATE): FILTER -- only if `record.set_by === 'coach'`. Otherwise return null (client edited own macros). Return `{ clientId: record.user_id, title: 'Macros Updated', body: 'Your coach updated your macro targets', route: '/macros' }`
       - `'weekly_checkins'` (UPDATE): FILTER -- only if `record.coach_response` is truthy AND `old_record?.coach_response` is falsy (newly added response, not edit). Return `{ clientId: record.client_id, title: 'Check-in Reviewed', body: 'Your coach responded to your weekly check-in', route: '/' }`
       - Default: return null

    4. Main handler with `Deno.serve`:
       - CORS preflight handling (OPTIONS -> 200)
       - Parse webhook payload from request body
       - Call getNotificationDetails -- if null, return `{ skipped: true }` with 200
       - Create admin Supabase client (service role key -- bypasses RLS to read device tokens for ANY user)
       - Query `device_tokens` table for `clientId` user: `.select('token').eq('user_id', details.clientId)`
       - If no tokens, return `{ sent: 0, reason: 'no_tokens' }` with 200
       - Loop through tokens, call `sendAPNs(device.token, { title, body, data: { route } })`
       - Count successes, return `{ sent: N }` with 200
       - Catch errors, return `{ error: message }` with 500

    5. Note on authentication: Database webhooks are sent from Supabase infrastructure itself, authenticated via the service_role key in the webhook header config. This function does NOT need to authenticate the caller like send-invite does -- it trusts the webhook source. However, security is maintained because only Supabase dashboard admins can create webhooks, and the function only reads (never writes) user data beyond device tokens.
