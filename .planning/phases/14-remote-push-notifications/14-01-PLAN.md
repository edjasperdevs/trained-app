  <action>
    1. Install the push plugin:
       ```
       npm install @capacitor/push-notifications
       npx cap sync
       ```

    2. Add PushNotifications config to capacitor.config.ts plugins section:
       ```typescript
       PushNotifications: {
         presentationOptions: ['badge', 'sound', 'alert'],
       },
       ```
       This shows native iOS banners when app is in foreground. Add `/// <reference types="@capacitor/push-notifications" />` at top.

    3. Add two delegate methods to AppDelegate.swift (inside the class body, after the existing `continue userActivity` method):
       ```swift
       func application(_ application: UIApplication,
         didRegisterForRemoteNotificationsWithDeviceToken deviceToken: Data) {
         NotificationCenter.default.post(
           name: .capacitorDidRegisterForRemoteNotifications, object: deviceToken)
       }

       func application(_ application: UIApplication,
         didFailToRegisterForRemoteNotificationsWithError error: Error) {
         NotificationCenter.default.post(
           name: .capacitorDidFailToRegisterForRemoteNotifications, object: error)
       }
       ```
       These are REQUIRED -- without them the Capacitor plugin never receives the APNs token from iOS.

    4. Add aps-environment to App.entitlements (inside the existing `<dict>`):
       ```xml
       <key>aps-environment</key>
       <string>development</string>
       ```
       Keep it as `development` for now -- it changes to `production` automatically when archiving for App Store/TestFlight.

    5. Create supabase/migrations/011_device_tokens.sql:
       ```sql
       CREATE TABLE device_tokens (
         id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
         user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
         token text NOT NULL,
         platform text NOT NULL CHECK (platform IN ('ios', 'android')),
         created_at timestamptz DEFAULT now(),
         updated_at timestamptz DEFAULT now(),
         UNIQUE(user_id, platform)
       );

       ALTER TABLE device_tokens ENABLE ROW LEVEL SECURITY;

       -- Users can read/write their own tokens only
       CREATE POLICY "Users manage own tokens"
         ON device_tokens FOR ALL
         USING (auth.uid() = user_id)
         WITH CHECK (auth.uid() = user_id);

       -- Index for Edge Function lookups by user_id
       CREATE INDEX idx_device_tokens_user_id ON device_tokens(user_id);
       ```

<task type="auto">
  <name>Task 2: Create push.ts module and wire token cleanup into signOut</name>
  <files>
    src/lib/push.ts

  <action>
    1. Create src/lib/push.ts with the following exports:

       **requestPushPermission(): Promise&lt;boolean&gt;**
       - Guard with `if (!isNative()) return false`
       - Call `PushNotifications.checkPermissions()` first
       - If `receive === 'granted'`, call `PushNotifications.register()` and return true
       - If `receive === 'denied'`, return false (permanently denied, must go to Settings)
       - If `receive === 'prompt'`, call `PushNotifications.requestPermissions()`
       - If granted after request, call `PushNotifications.register()` and return true
       - Otherwise return false

       **initPushListeners(userId: string): void**
       - Guard with `if (!isNative()) return`
       - `registration` listener: call `storeDeviceToken(userId, token.value)` -- upserts to device_tokens table via Supabase with `{ user_id, token, platform: 'ios', updated_at: new Date().toISOString() }` using `onConflict: 'user_id,platform'`
       - `registrationError` listener: call `captureError()` from sentry.ts -- do NOT show toast (user doesn't need to see this)
       - `pushNotificationActionPerformed` listener: extract `action.notification.data.route` and if present, set `window.location.href = data.route` (the React Router will pick this up -- this is simpler than importing navigate and works for cold start). This handles DEEP-02.
       - Do NOT add a `pushNotificationReceived` handler -- the `presentationOptions` config handles foreground display natively, no duplicate toast needed.

       **removeDeviceToken(userId: string): Promise&lt;void&gt;**
       - Delete from device_tokens where user_id matches and platform = 'ios'
       - Wrap in try/catch, captureError on failure (non-blocking -- sign out should not fail)

       Internal helper `storeDeviceToken(userId: string, token: string)`:
       - Import supabase client from `@/lib/supabase`
       - Upsert to device_tokens table

       Import pattern: Use direct imports (not barrel) -- `import { isNative } from '@/lib/platform'`, `import { supabase } from '@/lib/supabase'`, `import { captureError } from '@/lib/sentry'`

    2. Modify src/stores/authStore.ts signOut method:
       - Add `import { removeDeviceToken } from '@/lib/push'` at top
       - In signOut, BEFORE `await supabase.auth.signOut()`, call `await removeDeviceToken(get().user?.id ?? '')` -- must happen before session is cleared so the Supabase call has auth context
       - Wrap in try/catch so sign-out never fails due to token removal error
  </action>
