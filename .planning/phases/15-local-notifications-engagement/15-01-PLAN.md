1. Install dependencies:
   ```bash

   npx cap sync
   ```

2. Update `capacitor.config.ts`:
   - Add `/// <reference types="@capacitor/local-notifications" />` at the top with the other triple-slash references
   - Add `Badge` plugin config to the `plugins` object:
     ```typescript
     Badge: {
       persist: true,
       autoClear: false,
     },
     ```
   - Add `LocalNotifications` plugin config:
     ```typescript
     LocalNotifications: {
       smallIcon: 'ic_notification',
       iconColor: '#D4443B',
     },
     ```

3. Create `src/lib/notifications.ts`:
   - Import `LocalNotifications`, `Weekday` from `@capacitor/local-notifications`
   - Import `isNative` from `@/lib/platform`
   - Export `NOTIFICATION_IDS` as const object:
     - `DAILY_CHECKIN: 1`
     - `MACRO_LOGGING: 3`
     - `XP_CLAIM: 4`
     - `WEEKLY_CHECKIN: 5`
     - `STREAK_AT_RISK: 6`
     - (Workout days use IDs 20+dayOfWeek, i.e. 20-26)
   - Export `toCapWeekday` function: `(jsDay: number): Weekday => (jsDay + 1) as Weekday`
   - Export `cancelAllScheduled()`: if not native, return. Get pending via `LocalNotifications.getPending()`, cancel if any exist via `LocalNotifications.cancel(pending)`.
   - Export `scheduleAllNotifications(prefs: NotificationPreferences, workoutDays: number[])`:
     - If not native, return
     - Call `cancelAllScheduled()`
     - Build notifications array based on enabled prefs:
       a. Daily check-in (id: 1): `schedule: { on: { hour, minute } }`, title "Daily Check-in", body "Complete your daily check-in to maintain your streak.", extra: `{ route: '/' }`
       b. Workout (id: 20+day per workoutDays): `schedule: { on: { weekday: toCapWeekday(day), hour, minute } }`, title "Workout Day", body "Time to train. Let's go.", extra: `{ route: '/workouts' }`
       c. Macro logging (id: 3): `schedule: { on: { hour, minute } }`, title "Log Your Macros", body "Track your nutrition before the day ends.", extra: `{ route: '/macros' }`
       d. XP claim (id: 4): `schedule: { on: { weekday: Weekday.Sunday, hour, minute } }`, title "Weekly Reward Ready", body "Your weekly reward is ready to claim!", extra: `{ route: '/' }`
       e. Weekly check-in (id: 5): `schedule: { on: { weekday: Weekday.Saturday, hour, minute } }`, title "Weekly Check-in", body "Submit your weekly check-in before the week ends.", extra: `{ route: '/checkin' }`
       f. Streak-at-risk (id: 6): `schedule: { on: { hour, minute } }`, title "Don't Break Your Streak", body "You still have time to check in today.", extra: `{ route: '/' }`
     - If notifications.length > 0, call `LocalNotifications.schedule({ notifications })`
   - Use the `NotificationPreferences` type from remindersStore (import). Each pref has `enabled: boolean` and `time: { hour: number; minute: number }`.
  </action>

<task type="auto">
  <name>Task 2: Create badge module and extend reminders store with notification preferences</name>
  <files>
    src/lib/badge.ts
    src/stores/remindersStore.ts
  </files>
  <action>
1. Create `src/lib/badge.ts`:
   - Import `Badge` from `@capawesome/capacitor-badge`
   - Import `isNative` from `@/lib/platform`
   - Import `useXPStore` from `@/stores/xpStore` (direct import, not barrel, to avoid circular deps)
   - Export `updateBadge()`:
     - If not native, return
     - Call `computePendingActions()`, then `Badge.clear()` if count <= 0, else `Badge.set({ count })`
     - Wrap in try/catch -- non-blocking, fire-and-forget
   - Export `computePendingActions()`:
     - Check if daily check-in is pending: `useXPStore.getState().getTodayLog()` -- if no log or `!todayLog.checkIn`, count++
     - Check for unread coach response: read `localStorage.getItem('trained-latest-checkin')`, parse JSON, read `localStorage.getItem('trained-last-seen-coach-response')`. If parsed.status === 'reviewed' && parsed.reviewed_at && parsed.reviewed_at !== lastSeen, count++
     - Return count
   - Export `markCoachResponseSeen()`:
     - Read `trained-latest-checkin` from localStorage, parse, if `reviewed_at` exists, write it to `trained-last-seen-coach-response` in localStorage
     - Wrap in try/catch -- ignore parse errors

2. Extend `src/stores/remindersStore.ts`:
   - Add new exported types ABOVE the existing types:
     ```typescript
     export interface NotificationTimePreference {
       hour: number   // 0-23
       minute: number // 0-59
     }

     export interface NotificationPreferences {
       checkIn: { enabled: boolean; time: NotificationTimePreference }
       workout: { enabled: boolean; time: NotificationTimePreference }
       logMacros: { enabled: boolean; time: NotificationTimePreference }
       claimXP: { enabled: boolean; time: NotificationTimePreference }
       weeklyCheckIn: { enabled: boolean; time: NotificationTimePreference }
       streakProtection: { enabled: boolean; time: NotificationTimePreference }
     }
     ```
   - Add to the `RemindersStore` interface:
     - `notificationPreferences: NotificationPreferences`
     - `setNotificationPreference: (key: keyof NotificationPreferences, enabled: boolean) => void`
     - `setNotificationTime: (key: keyof NotificationPreferences, hour: number, minute: number) => void`
   - Add default `notificationPreferences` state in the store creator:
     ```typescript
     notificationPreferences: {
       checkIn: { enabled: true, time: { hour: 8, minute: 0 } },
       workout: { enabled: true, time: { hour: 7, minute: 0 } },
       logMacros: { enabled: true, time: { hour: 19, minute: 0 } },
       claimXP: { enabled: true, time: { hour: 10, minute: 0 } },
       weeklyCheckIn: { enabled: false, time: { hour: 10, minute: 0 } },
       streakProtection: { enabled: true, time: { hour: 20, minute: 0 } },
     },
     ```
     Note: weeklyCheckIn defaults to false (only relevant for coaching clients -- UI will handle visibility).
   - Implement `setNotificationPreference`:
     ```typescript
     setNotificationPreference: (key, enabled) => {
       set((state) => ({
         notificationPreferences: {
           ...state.notificationPreferences,
           [key]: { ...state.notificationPreferences[key], enabled },
         },
       }))
     },
     ```
   - Implement `setNotificationTime`:
     ```typescript
     setNotificationTime: (key, hour, minute) => {
       set((state) => ({
         notificationPreferences: {
           ...state.notificationPreferences,
           [key]: { ...state.notificationPreferences[key], time: { hour, minute } },
         },
       }))
     },
     ```
   - Export the new types from the stores barrel (`src/stores/index.ts`):
     `export type { NotificationTimePreference, NotificationPreferences } from './remindersStore'`

   IMPORTANT: Keep ALL existing store functionality (preferences, dismissedToday, shouldShow* methods, getActiveReminders, etc.) intact. The new `notificationPreferences` is a SEPARATE concern from the existing in-app `preferences` toggles. The in-app reminders (cards on home screen) use `preferences`. The push notification scheduling uses `notificationPreferences`.
  </action>
