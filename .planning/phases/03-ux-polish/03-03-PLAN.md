  **Create src/lib/haptics.ts:**

  A thin utility with feature detection and predefined vibration patterns. Uses `navigator.vibrate()` Web API.

  ```typescript
  const canVibrate = typeof navigator !== 'undefined' && 'vibrate' in navigator

  export const haptics = {
    /** Light tap - set completion, toggles */
    light: () => canVibrate && navigator.vibrate(10),

    /** Medium tap - action confirmed */
    medium: () => canVibrate && navigator.vibrate(25),

    /** Success pattern - workout complete, check-in, achievement unlock */
    success: () => canVibrate && navigator.vibrate([15, 50, 30]),

    /** Heavy tap - important milestone like XP claim */
    heavy: () => canVibrate && navigator.vibrate(50),

    /** Error buzz - something went wrong */
    error: () => canVibrate && navigator.vibrate([50, 30, 50]),
  }
  ```

  Key details:
  - `canVibrate` is evaluated once at module load. On iOS Safari and desktop browsers, `'vibrate' in navigator` is `false`, so all calls become no-ops.
  - Each method returns `boolean | false` (the `&&` short-circuits). This is fine -- callers don't check the return value.
  - Vibration durations are in milliseconds. Arrays alternate [vibrate, pause, vibrate, ...].
  - Keep total vibration under 100ms to feel like a tap, not a buzz.
