  <files>supabase/migrations/006_workout_programming.sql, supabase/schema.sql</files>
  <action>
Create `supabase/migrations/006_workout_programming.sql` with:

1. **workout_templates** table:
   - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
   - `created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL`
   - `updated_at TIMESTAMPTZ DEFAULT NOW() NOT NULL`
   - `coach_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE`
   - `name TEXT NOT NULL`
   - `exercises JSONB DEFAULT '[]'::jsonb NOT NULL`
   - Index on `coach_id`
   - RLS enabled with policy "Coaches can manage own templates" for ALL using `coach_id = auth.uid() AND EXISTS (SELECT 1 FROM profiles WHERE profiles.id = auth.uid() AND profiles.role = 'coach')`
   - `update_updated_at` trigger

2. **assigned_workouts** table:
   - `id UUID PRIMARY KEY DEFAULT uuid_generate_v4()`
   - `created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL`
   - `updated_at TIMESTAMPTZ DEFAULT NOW() NOT NULL`
   - `coach_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE`
   - `client_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE`
   - `template_id UUID REFERENCES workout_templates(id) ON DELETE SET NULL`
   - `date DATE NOT NULL`
   - `exercises JSONB NOT NULL` (snapshot of exercises at assignment time)
   - `notes TEXT`
   - `UNIQUE(client_id, date)` constraint
   - Indexes on `(client_id, date DESC)` and `coach_id`
   - RLS enabled with:
     - Coach policy for ALL using `coach_id = auth.uid() AND EXISTS (SELECT 1 FROM coach_clients WHERE coach_clients.coach_id = auth.uid() AND coach_clients.client_id = assigned_workouts.client_id AND coach_clients.status = 'active')`
     - Client SELECT policy using `client_id = auth.uid()`
   - `update_updated_at` trigger

3. **workout_logs.assignment_id** column:
   - `ALTER TABLE workout_logs ADD COLUMN assignment_id UUID REFERENCES assigned_workouts(id) ON DELETE SET NULL`
   - Partial index on `assignment_id WHERE assignment_id IS NOT NULL`

Then append these table definitions to `supabase/schema.sql` for documentation (the schema.sql file serves as a reference copy -- append after the existing content in the same style as existing tables).
