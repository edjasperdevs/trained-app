Rewrite all four components to use CVA (class-variance-authority) for variant definitions and cn() for class merging. Import `cva` from `class-variance-authority` and `cn` from `@/lib/cn`. Replace all template literal class concatenation with cn() calls.

**Button.tsx:**
- Define `buttonVariants = cva(base, { variants: { variant, size }, defaultVariants })`
- Base classes: `rounded transition-all duration-150 flex items-center justify-center gap-2`
- variant: primary (`bg-primary text-text-on-primary hover:bg-primary-hover font-heading uppercase tracking-widest font-semibold`), secondary (`bg-transparent border border-border text-text-primary hover:bg-secondary hover:border-secondary`), ghost (`bg-surface text-text-primary hover:bg-surface-elevated border border-border`), danger (`bg-error text-text-on-primary hover:opacity-90`)
- size: sm (`px-3 py-1.5 text-sm`), md (`px-5 py-2.5 text-base`), lg (`px-6 py-3.5 text-base`)
- Default variant: primary, default size: md
- Keep motion.button wrapper, whileTap/whileHover behavior, disabled logic
- Use `cn(buttonVariants({ variant, size }), fullWidth && 'w-full', disabled && 'opacity-50 cursor-not-allowed', className)` for className
- Export the `buttonVariants` alongside the component (useful for link-styled-as-button patterns)

**Card.tsx:**
- Define `cardVariants = cva(base, { variants: { variant, padding }, defaultVariants })`
- Base classes: `rounded-md transition-colors duration-150`
- variant: default (`bg-surface border border-border`), elevated (`bg-surface-elevated border border-border shadow-card`), subtle (`bg-surface/50 border border-border/50`)
- IMPORTANT: No glass effects, no backdrop-blur on any Card variant. Add `shadow-card` to elevated variant for premium depth.
- padding: none (``), sm (`p-3`), md (`p-4`), lg (`p-6`)
- Default variant: default, default padding: md
- Keep the motion.div/motion.button dynamic component logic based on onClick
- Use `cn(cardVariants({ variant, padding }), hover && 'card-hover cursor-pointer', onClick && 'text-left w-full', className)`

**ProgressBar.tsx:**
- Define `progressBarVariants = cva('h-full rounded-sm', { variants: { color }, defaultVariants })`
- color: primary (`bg-xp-bar`), secondary (`bg-secondary`), success (`bg-success`), warning (`bg-warning`), gradient (`bg-gradient-to-r from-primary to-primary-hover`)
- SIMPLIFY: Remove legacy color aliases (gold, cyan, green, purple). Map: gold->primary, cyan->primary, green->secondary, purple->secondary. Update the color type to only expose `'primary' | 'secondary' | 'success' | 'warning' | 'gradient'`.
- Define `progressTrackVariants = cva('w-full bg-xp-bar-bg rounded-sm overflow-hidden', { variants: { size } })`
- size: sm (`h-1`), md (`h-2`), lg (`h-3`), xl (`h-4`)
- Default color: primary, default size: md
- Use cn() for all class composition

**Toast.tsx (ToastContainer):**
- Define `toastVariants = cva('bg-surface border border-border rounded px-4 py-3 shadow-card flex items-center gap-3', { variants: { type } })`
- type: success (`border-l-[3px] border-l-success`), error (`border-l-[3px] border-l-error`), warning (`border-l-[3px] border-l-warning`), info (`border-l-[3px] border-l-info`)
- Keep the icon logic (Check, X, AlertTriangle, Info from lucide-react) but define it in a simple map object instead of a switch
- Keep AnimatePresence, motion.div wrapper, dismiss button
- Use cn() for all class composition

For all four components, ensure the Props interface is updated to match CVA variant types. Use `VariantProps<typeof xxxVariants>` from CVA to type the variant props.
