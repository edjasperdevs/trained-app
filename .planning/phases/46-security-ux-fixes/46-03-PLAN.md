Adjust the compliance calculation to handle recovery days (no workout scheduled) so users can achieve full compliance and unlock the share button.

**Current issue (lines 77-80):**
```typescript
// Full compliance = all 5 items checked (workout only counts if there was a workout scheduled)
// Per feature brief, share button only shows on "full 5/5 compliance days"
// On recovery days (no workout), user cannot achieve 5/5, so share button should NOT appear
const isFullCompliance = data.workout && data.protein && data.meal && data.steps && data.sleep
```

The comment acknowledges the problem: recovery days can't achieve 5/5 because there's no workout to complete.

**Fix:** Adjust `isFullCompliance` to check if a workout was scheduled. If no workout scheduled (recovery day), consider 4/4 (protein + meal + steps + sleep) as full compliance.

Replace lines 77-80 with:
```typescript
// Full compliance logic:
// - Training day (workout scheduled): 5/5 required (workout + protein + meal + steps + sleep)
// - Recovery day (no workout): 4/4 required (protein + meal + steps + sleep)
// This allows users on rest days to achieve shareable compliance
const hasWorkoutScheduled = todayWorkout !== null
const isFullCompliance = hasWorkoutScheduled
  ? (data.workout && data.protein && data.meal && data.steps && data.sleep)
  : (data.protein && data.meal && data.steps && data.sleep)
```

This change ensures:
1. On training days with a scheduled workout, users need 5/5 compliance
2. On recovery days (no workout in plan), users need 4/4 compliance (excluding workout item)
3. Share button appears appropriately in both scenarios

**Note:** `todayWorkout` is already available from `getTodayWorkout()` call on line 74.

**Alternative consideration:** The audit report suggests auto-checking the workout item on recovery days with a "Recovery Day" label. The approach above (4/4 on recovery days) is simpler and doesn't require UI changes, but if the user prefers the auto-check approach, that would require additional UI updates to show "Recovery Day" status in the workout compliance item.

Use the 4/4 approach for simplicity unless instructed otherwise during execution.
