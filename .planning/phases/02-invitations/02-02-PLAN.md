<task type="auto">
  <name>Task 2: Create send-invite Edge Function</name>
  <files>supabase/functions/send-invite/index.ts</files>
  <action>
Create `supabase/functions/send-invite/index.ts` implementing the full invite flow. Use the research example as the reference implementation. The function must:

1. **Handle CORS preflight:** Return 200 with corsHeaders for OPTIONS requests.

2. **Authenticate caller:** Create a Supabase client using the forwarded Authorization header and `SUPABASE_ANON_KEY`. Call `supabase.auth.getUser()` to verify the caller is authenticated.

3. **Verify coach role:** Create an admin client using `SUPABASE_SERVICE_ROLE_KEY` (bypasses RLS). Query `profiles` for the caller's role and email. Reject if not `coach`.

4. **Parse and validate email:** Read `{ email }` from request body. Trim and lowercase. Validate with regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`. Reject if invalid.

5. **Prevent self-invite:** Compare normalized email against coach's own email. Reject with "You cannot invite yourself".

6. **Check for existing user:**
   - Query `profiles` by email using admin client
   - If user exists:
     - Check if already in `coach_clients` for this coach. If so, reject "This user is already your client"
     - If not, INSERT into `coach_clients` directly (status: 'active') and return `{ success: true, action: 'added_directly' }`
   - If user does NOT exist, continue to invite flow

7. **Create/update invite (upsert):**
   - Use admin client to upsert into `invites` table with `onConflict: 'coach_id,email'`
   - Set coach_id, email, status='pending', expires_at = 7 days from now
   - On conflict (resend), update expires_at and status back to 'pending'
   - Select the token from the upserted row

8. **Send email via Resend:**
   - Read `RESEND_API_KEY` from `Deno.env.get()`. If missing, throw "Email service not configured"
   - Read `APP_URL` from env, default to 'https://app.trained.com'
   - POST to `https://api.resend.com/emails` with:
     - from: Read from env `EMAIL_FROM` with fallback to `'Trained <onboarding@resend.dev>'` (test domain)
     - to: [normalizedEmail]
     - subject: "You've been invited to Trained"
     - html: Branded invite email with signup link `${APP_URL}?invite=${invite.token}`
   - If Resend returns non-ok, log error and throw "Failed to send invite email"

9. **Return success:** `{ success: true, action: 'invite_sent' }`

10. **Error handling:** Catch all errors, return 400 with `{ error: message }` and corsHeaders.

**Email HTML template:** Create a clean, minimal branded HTML email:
- App name "Trained" prominently displayed
- Brief message: "Your coach has invited you to join Trained"
- CTA button linking to `${APP_URL}?invite=${token}`
- Expiry note: "This invite expires in 7 days"
- Simple inline CSS (no external stylesheets -- email clients strip them)
- Dark background (#1a1a2e), white text, red CTA button (#ef4444) matching the Trained brand

**Import pattern:** Use `https://esm.sh/@supabase/supabase-js@2` for the Supabase client (Deno-compatible ESM import).
