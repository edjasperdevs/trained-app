  <action>
Read the current PrivacyInfo.xcprivacy file. The NSPrivacyCollectedDataTypes array is currently empty.

Populate NSPrivacyCollectedDataTypes with all data categories the app collects, based on AUDIT_REPORT.md Section 5.3:

1. **Health & Fitness** — Steps (HealthKit), sleep duration (HealthKit), workout logs, body weight

   - NSPrivacyCollectedDataType: NSPrivacyCollectedDataTypeOtherUserContent

   - Used for tracking: false

   - NSPrivacyCollectedDataType: NSPrivacyCollectedDataTypeProductInteraction

   - Purposes: Analytics

   - NSPrivacyCollectedDataType: NSPrivacyCollectedDataTypePurchaseHistory
   - Linked to user: true
   - Used for tracking: false

   - NSPrivacyCollectedDataType: NSPrivacyCollectedDataTypePhotos

<task type="auto">
  <name>Task 2: Update App.entitlements to production APNS environment</name>
  <files>ios/App/App/App.entitlements</files>
  <action>
Read ios/App/App/App.entitlements. Currently aps-environment is set to "development" (line 6).

Change the aps-environment value from "development" to "production":

```xml
<key>aps-environment</key>
<string>production</string>
```

This ensures push notifications route through Apple's production APNS servers when the app is distributed via the App Store. The development environment is only valid for local Xcode builds and TestFlight internal testing.

Leave all other entitlement keys unchanged:
- com.apple.developer.applesignin
- com.apple.developer.associated-domains
- com.apple.developer.healthkit
- com.apple.developer.in-app-payments

Reference AUDIT_REPORT.md Section 6.2 item 2: "The aps-environment value of 'development' will cause push notifications to fail in the production App Store build."
  </action>
  <verify>
    <automated>grep 'aps-environment' ios/App/App/App.entitlements -A 1 | grep 'production'</automated>
    <manual>Open App.entitlements in Xcode and confirm aps-environment shows "production"</manual>
    <sampling_rate>run after this task commits, before next task begins</sampling_rate>
  </verify>
  <done>App.entitlements has aps-environment set to "production" — production builds will receive push notifications via APNS production servers</done>
</task>
