```markdown
# WellTrained App Store Review Notes

## Demo Account
Email: reviewer@welltrained.fitness
Password: [CREATE AND INSERT SECURE PASSWORD]

Note: Demo account has premium subscription enabled for full feature access.

## Testing In-App Purchases

1. Sandbox account is pre-configured in RevenueCat
2. On Paywall screen, tap "Get Premium" (monthly or annual)
3. Sandbox purchase flow will complete with test credentials
4. After purchase, premium features unlock:
   - Premium archetypes (Himbo, Brute, Pup, Bull)
   - Weekly Protocol Orders
   - Avatar stages 3-5

If sandbox purchases fail (known Apple issue), please use the demo account which has premium enabled.

## Testing HealthKit

1. App requests HealthKit permission after onboarding (soft-ask pattern first)
2. Grant "Steps" and "Sleep" read access when prompted
3. Navigate to Home screen to see health data displayed
4. Earning 10,000+ steps awards +10 DP
5. Sleeping 7+ hours awards +10 DP

**Important:** HealthKit data requires physical device (not available on simulator).

**Note on Sleep Data:** Due to iOS HealthKit API limitations, sleep duration requires manual entry. Steps are auto-populated from Apple Health.

## HealthKit Data Usage

- App reads step count and sleep duration from Apple Health
- Data is used ONLY to award Discipline Points for healthy habits
- No health data is transmitted off-device or to third parties
- No advertising or analytics uses health data
- Data collection is declared in PrivacyInfo.xcprivacy

## App Functionality Overview

1. **Onboarding**: User enters basic profile info and selects archetype
2. **Home Screen**: Shows rank progress, obedience streak, daily Protocol Orders
3. **Workouts**: Log workout completion to earn +50 DP
4. **Macros**: Track daily meals (+15 DP each, max 3) and protein target (+25 DP)
5. **Settings**: Manage profile, subscription, export data

## Known Behaviors

- First launch shows onboarding flow (skip via demo account)
- Premium subscription unlocks immediately after purchase
- Fresh Start modal appears once for users upgrading from V1 (not visible to new users)
```

This task requires human action in Xcode and App Store Connect.

**Pre-submission checklist (all should be complete from prior plans):**
- [x] Data migration implemented (Plan 01)
- [x] PrivacyInfo.xcprivacy updated (Plan 02)
- [x] AASA file has real Team ID (Plan 02)
- [x] Metadata prepared (Task 1 above)

**Step 1: Archive and Upload to TestFlight**

1. Open Xcode: `open ios/App/App.xcworkspace`
2. Select target "App" and device "Any iOS Device"
3. Product -> Archive
4. When archive completes, click "Distribute App"
5. Select "App Store Connect" -> "Upload"
6. Follow prompts, select distribution profile
7. Wait for build processing in App Store Connect (5-30 minutes)

**Step 2: Test on TestFlight**

1. Install TestFlight app on physical iOS device
2. Accept TestFlight invitation (if first time)
3. Install WellTrained build
4. Verify:
   - Onboarding flow works
   - Premium subscription purchase works (sandbox)
   - HealthKit permission request appears
   - Steps display if granted
   - Fresh Start modal appears if you have old V1 data

**Step 3: Configure App Store Connect**

1. Go to App Store Connect -> My Apps -> WellTrained
2. Fill in metadata from .planning/app-store/metadata.md
3. Upload screenshots (see screenshots.md for specs)
4. Add review notes from .planning/app-store/review-notes.md
5. Create demo account (reviewer@welltrained.fitness) with premium enabled
6. Set up IAP products if not already configured:
   - welltrained.premium.monthly
   - welltrained.premium.annual
7. Mark IAP products as "Ready to Submit"

**Step 4: Submit for Review**

1. Verify all fields are complete (green checkmarks)
2. Answer Export Compliance (HTTPS only = Yes, exempt)
3. Answer Content Rights (Yes, you have rights)
4. Answer Advertising Identifier (No, not using IDFA)
5. Click "Submit for Review"
